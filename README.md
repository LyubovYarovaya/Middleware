# KeyCRM ➡️ GA4 & Google Ads Middleware

Цей мікросервіс (Middleware) призначений для надійного перехоплення вебхуків з KeyCRM (як для класичних замовлень, так і для карток у воронках/чатах) та їхньої передачі до **Google Analytics 4** (Measurement Protocol) та в **Google Таблиці** (для офлайн-конверсій Google Ads).

## 🚀 Основні можливості
- **Обробка 100% вебхуків KeyCRM**: Скрипт розуміє різницю між `Order` (Класичне замовлення) та `Pipeline Card` (Картка у воронці або чат). Система уникає колізій ID за допомогою порівняння часових міток.
- **Відправка подій у GA4 (Measurement Protocol)**:
  - `lead` (якщо статус = 1, або прийшов лід з воронки)
  - `purchase` (статуси у KeyCRM 23, 24)
  - `refund` (статус 19 - скасування)
- **Підтримка UTM-міток**: Витягує `utm_source`, `utm_medium`, `utm_campaign` (із вбудованих полів або кастомних полів) і передає їх як `user_properties`.
- **Захист від дублікатів (Deduplication)**: Використовує Redis. Забороняє повторну відправку `lead` чи `purchase` для одного й того самого ID впродовж 30 днів.
- **Генерація `client_id`**: Витягує `ga_client_id` або `OR_1004`. Якщо його немає (чи дорівнює `unknown-client`), скрипт автоматично генерує та закріплює за цим лідом випадковий UUID.
- **Підготовка офлайн-конверсій для Google Ads**: Записує події прямо у Google Таблицю на аркуш `GoogleAds` у форматі, готовому для імпорту в рекламний кабінет.

## 📊 Структура Google Таблиць
Додаток надсилає POST-запити до вашого Google Apps Script, який формує три аркуші:
1. **`Webhooks`** — зберігає абсолютно всі вебхуки, що надходять, їхні статуси та `full_json` (структурований об'єкт).
2. **`GA4_Measurement`** — зберігає лог відправки в Google Analytics, включно з посиланням на подію, статусом (напр. `Sent OK (HTTP 204)`) і заблокованими дублями (`Skipped (Deduplicated)`).
3. **`GoogleAds`** — таблиця офлайн-конверсій:
   - `conversion_name` (`Звонки`, `Меседжеры`, `Купити в один клік`, `Оплата частинами`, `Other`)
   - `status_lead` (значення з поля `LD_1015` або `status_lead`, якщо заповнене).
   - `conversion_event_time` (час у вашому часовому поясі)
   - `gclid`, `currency_code`, `order_id`
   - `gbraid`, `wbraid`, `conversion_value`, `User Agent`, `IP-адрес`, `Session attributes`

## ✨ Кастомні поля та Воронки (Pipelines)
Скрипт має вбудовану гнучкість для лідів:
* Якщо `checkout_type` (або `OR_1003`) містить `"Купити в один клік"` ➡️ Конверсія: `Купити в один клік`.
* Якщо `checkout_type` містить `"Оплата частинами ПриватБанк"` чи `"МоноБанк"` ➡️ Конверсія: `Оплата частинами`.
* Якщо `pipeline_id === 1` ➡️ Конверсія: `Звонки`.
* Якщо `pipeline_id === 2` ➡️ Конверсія: `Меседжеры`.

🔥 **Особливість для Воронок 1 та 2**: Для події `lead` у GA4 їх стандартні атрибути замінюються на дані з кастомних полів:
* `lead_source` ➡️ `LD_1012` (`lead_source_v`)
* `lead_handled` ➡️ `LD_1013` (`lead_handled_v`)
* `checkout_type` ➡️ `LD_1015` (`status_lead`)
(Примітка: масиви у кастомних полях автоматично "розпаковуються" у звичайний текст через кому: `["raw_leads"] -> "raw_leads"`).

## ⚙️ Налаштування оточення (.env)
Обов'язкові змінні:
- `KEYCRM_API_KEY` — Ваш API-ключ для доступу до запитів `/v1/order/...` та `/v1/pipelines/cards/...`
- `GA4_MEASUREMENT_ID` — Ідентифікатор потоку даних GA4 (напр., `G-XXXXXXXXXX`)
- `GA4_API_SECRET` — Секретний ключ API з GA4.
- `REDIS_URL` — Підключення до бази даних Redis (надається Railway).
- `GOOGLE_SHEET_WRITER_URL` — Макро-URL вашого Google Apps Script (закінчується на `/exec`).

## 🛠 Запуск і Деплой
Система використовує `Node.js` та `BullMQ` (для черг на випадок збоїв GA4). Запускається командами:
\`\`\`bash
npm install
npm run build
npm start
\`\`\`
*У Railway налаштовано автоматичний деплой. Підключений плагін Redis забезпечує безперебійну роботу черг подій та системи відлову дублювань (deduplication).*
